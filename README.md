## Core Features

### Protocol Prediction Markets
Curated binary (YES / NO) markets centered on objectively measurable real-world events — including macroeconomic developments, public data releases, and technological milestones. All markets follow predefined resolution criteria and are settled using transparent, independently verifiable data provided through decentralized oracle systems, ensuring consistency, auditability, and integrity of outcomes.

### AI-Powered Market Intelligence
An embedded AI agent interface (powered by Google Gemini via Vercel AI SDK) delivers real-time, context-aware intelligence directly within the trading terminal. It synthesizes live data, event context, and probabilistic outcome reasoning to help users interpret markets and evaluate scenarios through a native conversational layer.

AI Agent V2.0 evolves this into a personalized trading co-pilot — a tailored assistant that can execute, customize, and manage trades on behalf of the user in seconds. It adapts to individual strategies, automates workflows, and streamlines decision-making from insight to execution within a unified interface.

### Crypto Hourly Price Markets
Real-time price prediction markets for **BTC**, **ETH**, and **SOL** that roll over every hour. Strike prices are anchored to live oracle feeds, creating a continuous stream of short-duration opportunities for traders who want fast, high-frequency exposure to crypto price movements.

### Permissionless Markers (Creator Markets)
Any verified trader can create their own prediction market on any topic. A tiered eligibility system — based on on-chain trading history, account age, and volume thresholds — ensures market quality. Creators earn a share of all trading fees generated by their market.

### Market Order and Limit Order Trading
Full-featured order support:
- **Market orders** — instant execution at current YES/NO prices
- **Limit orders** — set your target price; fill when the market moves to you
- Resilient trade recovery with local pending queue to prevent loss on reconnection

### Copy Trading
Follow the platform's top-performing "Gurus" and automatically mirror their positions with a single tap. The V2 engine supports:
- Real-time intent broadcasting when a Guru places a trade
- Follower queue with asynchronous transaction submission
- Relationship management (follow / unfollow) with on-chain settlement parity

### Leaderboard and Portfolio Analytics
Comprehensive performance tracking with PnL rankings, win rate calculation, total volume, and historical position data. Profiles are on-chain addressable — share your trading record with anyone via wallet address.

### Rewards and Points System
A multi-layered incentive engine that distributes points for:
- Market trading activity
- Social actions (X/Twitter verification, Discord linking, Telegram binding)
- Referral conversions

Points power the platform's long-term loyalty program and future airdrop eligibility.

### Creator Fee System
UGC market creators earn on-chain HELM token rewards proportional to the volume their market generates. A tiered fee schedule scales creator earnings with market success. Withdrawals are processed on-chain via SPL token transfer from the protocol treasury.

### Social Layer — The Square
A Twitter-like social feed where traders share market calls, post analysis, and attach trade cards to posts. Copy-trade intents surface natively in the feed, letting followers act on Guru trades with one tap.

### Social Identity Integrations
- **X (Twitter)** — OAuth verification with badge display
- **Discord** — Server membership verification
- **Telegram** — Bot-based account binding with push notifications for trade events and market settlements

### Robust Settlement and Keeper Engine
An automated keeper system polls open markets for settlement eligibility. When a market reaches its resolution criteria, the keeper executes bulk settlement transactions on-chain, distributing HELM payouts to all winning positions atomically.

## Platform Architecture

```
+-----------------------------------------------------------+
|                       helm.trading                        |
|                Next.js 13 App Router (Vercel)             |
+---------------+----------------------+--------------------+
|   Frontend    |     API Routes       |   Background Jobs  |
|   (React /    | /api/trade/save      |   Keeper (Cron)    |
|   Tailwind)   | /api/markets/create  |   Settlement Bot   |
|               | /api/copy/*          |   UGC Leaderboard  |
|               | /api/settle/*        |   Copy Reconcile   |
|               | /api/keeper          |                    |
+---------------+----------+----------+--------------------+
                           |
          +----------------+----------------+
          |                                 |
   Supabase (Postgres)             Solana Blockchain
   - markets                       - SPL Token (HELM V3)
   - orders                        - Treasury Wallet
   - profiles                      - On-chain settlements
   - ugc_markets                   - Creator fee withdrawals
   - copy_trading_*                - Multi-RPC pool (fallback)
   - ugc_rate_limits
```

### Data Flow

```
User Wallet (Phantom / Backpack / Solflare)
        |
        v
  Trading Interface  -->  Sign and Submit Transaction
        |                              |
        |                    Solana RPC Pool (w/ fallback)
        |                              |
        v                              v
  Trade API          <------  TX Confirmation
        |
        v
  Supabase orders table  -->  Keeper Cron  -->  Bulk Settlement
        |
        v
  Profile stats update  -->  Rewards engine  -->  Points accrual
```

## Getting Started

### Prerequisites

- Node.js 18+
- A Solana devnet wallet (Phantom recommended)
- Supabase project with the schema migrations applied
- Solana devnet RPC endpoint

### Installation

```bash
git clone https://github.com/Helm-Team/HELM-Code.git
cd HELM-Code
npm install
cp .env.example .env.local
npm run dev
```

Open [http://localhost:3000](http://localhost:3000) in your browser.

## Environment Variables

Copy `.env.example` to `.env.local`. Never commit real credentials to version control.

### Core (Required)

| Variable | Description |
|---|---|
| `NEXT_PUBLIC_SUPABASE_URL` | Supabase project URL |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | Supabase anon public key |
| `SUPABASE_SERVICE_ROLE_KEY` | Supabase service role key (server-side only) |
| `NEXT_PUBLIC_SOLANA_RPC_URL` | Primary Solana RPC endpoint |
| `NEXT_PUBLIC_TEST_USDC_MINT` | HELM token mint address |
| `NEXT_PUBLIC_ADMIN_WALLET_ADDRESS` | Treasury wallet public key |
| `CRON_SECRET` | Bearer token for cron job authentication |

### Social Integrations (Optional)

| Variable | Description |
|---|---|
| `NEXT_PUBLIC_X_CLIENT_ID` | X (Twitter) OAuth 2.0 client ID |
| `NEXT_PUBLIC_DISCORD_CLIENT_ID` | Discord OAuth client ID |
| `HELM_TG_BOT_TOKEN` | Telegram bot token |
| `GOOGLE_GENERATIVE_AI_API_KEY` | Google Gemini API key (AI chat feature) |
