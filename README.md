## Core Features
### рҹҺҜ Protocol Prediction Markets
Curated binary (YES / NO) markets on high-signal real-world events вҖ” from Federal Reserve decisions and election outcomes to crypto milestones and scientific breakthroughs. Each market resolves to a definitive on-chain outcome settled against verifiable oracle data.
### вҸұпёҸ Crypto Hourly Price Markets
Real-time price prediction markets for **BTC**, **ETH**, and **SOL** that roll over every hour. Strike prices are anchored to live oracle feeds, creating a continuous stream of short-duration opportunities for traders who want fast, high-frequency exposure to crypto price movements.
### рҹҸ—пёҸ UGC Markets (User-Generated Content)
Any verified trader can create their own prediction market on any topic. A tiered eligibility system вҖ” based on on-chain trading history, account age, and volume thresholds вҖ” ensures market quality. Creators earn a share of all trading fees generated by their market.
### рҹ’ё Market Order & Limit Order Trading
Full-featured order support:
- **Market orders** вҖ” instant execution at current YES/NO prices
- **Limit orders** вҖ” set your target price; fill when the market moves to you
- Resilient trade recovery with local pending queue to prevent loss on reconnection
### рҹ“Ӣ Copy Trading
Follow the platform's top-performing "Gurus" and automatically mirror their positions with a single tap. The V2 engine supports:
- Real-time intent broadcasting when a Guru places a trade
- Follower queue with asynchronous transaction submission
- Relationship management (follow / unfollow) with on-chain settlement parity
### рҹҸҶ Leaderboard & Portfolio Analytics
Comprehensive performance tracking with PnL rankings, win rate calculation, total volume, and historical position data. Profiles are on-chain addressable вҖ” share your trading record with anyone via wallet address.
### рҹҺҒ Rewards & Points System
A multi-layered incentive engine that distributes points for:
- Market trading activity
- Social actions (X/Twitter verification, Discord linking, Telegram binding)
- Referral conversions
Points power the platform's long-term loyalty program and future airdrop eligibility.
### рҹ’° Creator Fee System
UGC market creators earn on-chain HELM token rewards proportional to the volume their market generates. A tiered fee schedule scales creator earnings with market success. Withdrawals are processed on-chain via SPL token transfer from the protocol treasury.
### рҹӨ– AI-Powered Market Intelligence
An embedded AI chat interface (powered by **Google Gemini** via Vercel AI SDK) provides traders with real-time market analysis, event context, and outcome reasoning вҖ” directly inside the trading terminal.
### рҹ“Ў Social Layer вҖ” The Square
A Twitter-like social feed where traders share market calls, post analysis, and attach trade cards to posts. Copy-trade intents surface natively in the feed, letting followers act on Guru trades with one tap.
### рҹ”— Social Identity Integrations
- **X (Twitter)** вҖ” OAuth verification with badge display
- **Discord** вҖ” Server membership verification
- **Telegram** вҖ” Bot-based account binding with push notifications for trade events and market settlements
### рҹӣЎпёҸ Robust Settlement & Keeper Engine
An automated keeper system polls open markets for settlement eligibility. When a market reaches its resolution criteria, the keeper executes bulk settlement transactions on-chain, distributing HELM payouts to all winning positions atomically.

## Getting Started
### Prerequisites
- Node.js 18+
- A Solana devnet wallet (Phantom recommended)
- Supabase project with the schema migrations applied
- Solana devnet RPC endpoint
### Installation
```bash
# Clone the repository
git clone https://github.com/Helm-Team/HELM-Code.git
cd HELM-Code
# Install dependencies
npm install
# Copy the environment template
cp .env.example .env.local
# Start the development server
npm run dev
Open http://localhost:3000 in your browser.

Environment Variables
Copy .env.example to .env.local. Never commit real credentials to version control.
```
