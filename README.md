## Core Features

### 🎯 Protocol Prediction Markets
Curated binary (YES / NO) markets on high-signal real-world events — from Federal Reserve decisions and election outcomes to crypto milestones and scientific breakthroughs. Each market resolves to a definitive on-chain outcome settled against verifiable oracle data.

### ⏱️ Crypto Hourly Price Markets
Real-time price prediction markets for **BTC**, **ETH**, and **SOL** that roll over every hour. Deterministic strike prices are generated from a seeded algorithm anchored to the current market price, creating a continuous stream of short-duration opportunities with predictable mechanics.

### 🏗️ UGC Markets (User-Generated Content)
Any verified trader can create their own prediction market on any topic. A tiered eligibility system — based on on-chain trading history, account age, and volume thresholds — ensures market quality. Creators earn a share of all trading fees generated by their market.

### 💸 Market Order & Limit Order Trading
Full-featured order book supporting:
- **Market orders** — instant execution at current YES/NO prices
- **Limit orders** — set your target price; fill when the market moves to you
- LocalStorage-backed pending queue for resilient trade recovery across reconnects

### 📋 Copy Trading
Follow the platform's top-performing "Gurus" and automatically mirror their positions with a single tap. The V2 engine supports:
- Real-time intent broadcasting when a Guru places a trade
- Follower queue with asynchronous transaction submission
- Relationship management (follow / unfollow) with on-chain settlement parity

### 🏆 Leaderboard & Portfolio Analytics
Comprehensive performance tracking with PnL rankings, win rate calculation, total volume, and historical position data. Profiles are on-chain addressable — share your trading record with anyone via wallet address.

### 🎁 Rewards & Points System
A multi-layered incentive engine that distributes points for:
- Market trading activity
- Social actions (X/Twitter verification, Discord linking, Telegram binding)
- Referral conversions
- Faucet interactions (devnet)

Points power the platform's long-term loyalty program and future airdrop eligibility.

### 💰 Creator Fee System
UGC market creators earn on-chain HELM token rewards proportional to the volume their market generates. A tiered fee schedule (`Tier 1 → Tier 3`) scales creator earnings with market success. Withdrawals are processed on-chain via SPL token transfer from the protocol treasury.

### 🤖 AI-Powered Market Intelligence
An embedded AI chat interface (powered by **Google Gemini** via Vercel AI SDK) provides traders with real-time market analysis, event context, and outcome reasoning — directly inside the trading terminal.

### 📡 Social Layer — The Square
A Twitter-like social feed where traders share market calls, post analysis, and attach trade cards to posts. Copy-trade intents surface natively in the feed, letting followers act on Guru trades with one tap.

### 🔗 Social Identity Integrations
- **X (Twitter)** — OAuth verification with badge display
- **Discord** — Server membership verification
- **Telegram** — Bot-based account binding with push notifications for trade events and market settlements

### 🛡️ Robust Settlement & Keeper Engine
An automated keeper system polls open markets for settlement eligibility. When a market reaches its resolution criteria, the keeper executes bulk settlement transactions on-chain, distributing HELM payouts to all winning positions atomically.

## Getting Started

### Prerequisites

- Node.js 18+
- A Solana devnet wallet (Phantom recommended)
- Supabase project with the schema migrations applied
- Solana devnet RPC endpoint

### Installation

```bash
# Clone the repository
git clone https://github.com/Helm-Team/HELM-Code.git
cd HELM-Code

# Install dependencies
npm install

# Copy the environment template
cp .env.example .env.local
# Edit .env.local with your credentials (see Environment Variables below)

# Apply Supabase migrations
# Run each file in supabase/migrations/ in chronological order
# via your Supabase project SQL editor or the Supabase CLI

# Start the development server
npm run dev
```

Open [http://localhost:3000](http://localhost:3000) in your browser.

### Useful Scripts

```bash
npm run dev              # Start dev server (Next.js)
npm run build            # Production build
npm run typecheck        # TypeScript type checking (no emit)
npm run verify-ugc-rules # Validate UGC market eligibility rules
npm run update-token-meta # Update HELM token metadata on-chain
npm run create-helm-v3   # Deploy a new HELM V3 token mint
```

## Environment Variables

Copy `.env.example` to `.env.local` and fill in the values below.

### Core (Required)

| Variable | Description |
|----------|-------------|
| `NEXT_PUBLIC_SUPABASE_URL` | Supabase project URL |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | Supabase anon public key |
| `SUPABASE_SERVICE_ROLE_KEY` | Supabase service role key (server-side only) |
| `NEXT_PUBLIC_SOLANA_RPC_URL` | Primary Solana RPC endpoint |
| `NEXT_PUBLIC_SOLANA_RPC_URLS` | Comma-separated fallback RPCs for pool |
| `NEXT_PUBLIC_TEST_USDC_MINT` | HELM token mint address (`4wMp7...` for V3) |
| `NEXT_PUBLIC_ADMIN_WALLET_ADDRESS` | Treasury / admin wallet public key |
| `ADMIN_PRIVATE_KEY` | Treasury wallet private key (server-side, cron jobs) |
| `CRON_SECRET` | Bearer token for authenticating cron job endpoints |

### UGC Markets

| Variable | Description | Default |
|----------|-------------|---------|
| `UGC_CREATE_REQUIRE_SIGNATURE` | Require wallet signature on market creation | `true` |
| `UGC_MARKET_MIN_ACCOUNT_AGE_DAYS` | Minimum account age to create a market | `7` |
| `UGC_CREATE_MIN_LIFETIME_ORDERS` | Minimum lifetime trades required | `25` |
| `UGC_MARKET_CREATE_PER_DAY` | Markets per wallet per day | `3` |
| `UGC_CREATOR_FEE_ENABLED` | Enable creator fee accrual | `true` |

### Social Integrations (Optional)

| Variable | Description |
|----------|-------------|
| `NEXT_PUBLIC_X_CLIENT_ID` | X (Twitter) OAuth 2.0 client ID |
| `X_CLIENT_SECRET` | X OAuth client secret |
| `NEXT_PUBLIC_DISCORD_CLIENT_ID` | Discord OAuth client ID |
| `DISCORD_CLIENT_SECRET` | Discord OAuth client secret |
| `HELM_TG_BOT_TOKEN` | Telegram bot token |
| `NEXT_PUBLIC_TG_BOT_NAME` | Telegram bot username |
| `GOOGLE_GENERATIVE_AI_API_KEY` | Google Gemini API key (AI chat feature) |
| `NEXT_PUBLIC_WALLETCONNECT_PROJECT_ID` | WalletConnect project ID |
